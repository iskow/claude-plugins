# Sources map — which official source answers what

Order of preference: official/primary source → recognized standards body → reputable secondary. Always note the version/date shown on the page.

## Relativity / RelativityOne

| Question type | Source |
|---|---|
| Features, settings, how-tos, limits | [help.relativity.com](https://help.relativity.com/) — versioned docs; note the release |
| Workflows, known issues, community answers | [community.relativity.com](https://community.relativity.com/) |
| Training courses, study plans | [learning.relativity.com](https://learning.relativity.com/) |
| Certifications (exams, updates, registration) | [relativity.com/resources/certification](https://www.relativity.com/resources/certification/) + the official Study Resources PDFs on help.relativity.com |
| Release notes / what changed | help.relativity.com release notes + [Exam Update Schedule](https://www.relativity.com/resources/certification/exam-updates/) |

## eDiscovery framework and law

| Question type | Source |
|---|---|
| EDRM model, IGRM, metrics, datasets | [EDRM.net](https://edrm.net/) (CC BY 4.0 — cite "EDRM (edrm.net)") |
| FRCP rule text (26, 34, 37(e), 45) | [law.cornell.edu/rules/frcp](https://www.law.cornell.edu/rules/frcp) or uscourts.gov |
| Best-practice principles, commentary | [The Sedona Conference](https://thesedonaconference.org/) — Sedona Principles, commentaries |
| TAR terminology | Grossman–Cormack, *Glossary of Technology-Assisted Review* (Fed. Cts. L. Rev.) |
| Case law on eDiscovery disputes | Prefer the opinion text; secondary: ediscoverytoday.com, ComplexDiscovery, EDRM blog |

## Certifications

| Question type | Source |
|---|---|
| ACEDS CEDS exam, packages, recertification | [aceds.org/ceds-certifications](https://aceds.org/ceds-certifications/) + CEDS Candidate Handbook (PDF) |
| ACEDS training certificates (eDEx, eDTech, eDAdv) | aceds.org training pages |
| Relativity exam structure and topics | Official Study Resources PDF per exam (help.relativity.com/Relativity/Certification/) — dated; re-fetch before relying on structure |

## Rules of engagement

- Version-specific or how-to detail → always doc-check, never memory.
- Blogs and vendor marketing are context, not authority — fine for "what are people doing", not for "what does the feature do".
- Exam prep dumps (Udemy question banks, killexams, etc.) → never use; both Relativity and ACEDS warn against third-party materials, and they're frequently wrong.
- If two official sources conflict, cite both and flag the conflict.
